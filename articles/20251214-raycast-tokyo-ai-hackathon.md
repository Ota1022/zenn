本記事は[Raycast Advent Calendar 2025](https://qiita.com/advent-calendar/2025/raycast)の19日目の記事です。

2025/10/23〜10/31にかけて Raycast Community Japan 主催のアイデアソン・ハッカソン・Meetup に参加しました。
すでに2ヶ月近く経ってしまったのですが、備忘録としてアイデアソンでのブレストから実際に Raycast Extension を作るまでの流れ、Meetup でのコミュニティとの交流を振り返ろうと思います。

## 1. 参加前

年初に転職してセットアップを始めた PC にクリップボード系のアプリを入れようと思っていたところ、Raycast がアツくて他の作業効率化機能も充実しているということを times で見かけてインストールしました。とはいえ実際に使っていたのは Clipboard History ばかりで、他の標準機能や Extension はなかなか活用できずにいました。

半年以上が経ったある日、Raycast・Anthropic・Vercel・Supabase がコラボしてハッカソンを開催するという情報を X で見かけ、豪華な座組にかなり驚きました。
いまいち活用できていない Raycast を使いこなす絶好の機会だと感じ、チームが組めなくてもソロで参加すればいいかと思い参加登録しました。

## 2. 参加したイベント

ハッカソンの前後にも Raycast Community Japan 主催でイベントが予定されており、せっかくなので業務調整をして 3 つとも参加することにしました。

* **10/23（木） アイデアソン_TOKYO AI Hackathon プレイベント**
  参加者有志で日常的な課題と解決策を発散・共有
* **10/30（木） Tokyo AI Hackathon**
  Claude Code を駆使した1日での Raycast Extension 作成とピッチ
* **10/31（金） Raycast Meetup Japan #2 with Raycast**

## 3. 10/23 アイデアソン_TOKYO AI Hackathon プレイベント

イベントページ: [https://raycast.connpass.com/event/371015/](https://raycast.connpass.com/event/371015/)

プレイベントは目黒の primeNumber オフィスで行われました。

### 3.1 プレイベントの目的

参加者は 10 人程度で、Raycast 初心者から使いこなしている人まで集まりました。
このプレイベントは、ハッカソンで作る具体的なプロダクトに落とし込む前の段階として、自分が日常的に繰り返していることをスピード重視で付箋に書き出していくという形式でした。

業務の枠にとらわれず日々の行動の中で「毎回やってるな〜」「面倒だな〜」と感じることを書き出し、付箋を同じテーブルの参加者に共有して意見を交わしながら、アイデアを広げていきました。

![プレイベントの様子](/images/20251214/1_preevent.jpg)

### 3.2 ブレストで出てきたアイデア

自分のテーブルでは以下のような意見が挙がりました(以下、一例)。

* AWS などのコマンドを忘れて調べ直す
* 複数の業務PCにまたがったカレンダー管理が面倒
* Git ブランチの prefix を一日に何回も書いている

どれも共感できる課題ばかりで、自分もエンジニアとしてもっと怠惰であるべきというか、見逃していたけどもっと効率化できそうなところはあるなという発見がありました。

ハッカソンでは自分が困っていることを解決したいと思っていたので、自分がメインで担当しがちな作業の手順を誰かに聞かれたときに、うまく言語化・整理してフォーマットに落としてくれるツールがあればいいなと思いました。「 Clipboard History を元に手順書を作る」という方向性で行こうという方針を固められたので非常に収穫でした。

### 3.3 Raycast の Tips を知る

プレイベントではアイデア出しだけでなく、Raycast Community Japan のメンバーから実践的な使い方も学ぶことができました。

私は本当に初心者だったので Hotkey の設定と Snippets を知らなかったのですが、よく使う Extension には個別に Hotkey を割り当てることで直接起動できるようになり、作業がスムーズになりました。また、Snippets 機能を使えば、コマンドのテンプレートなどを登録して呼び出せるため、日常的な作業の効率が大幅に向上します。

実際に使いこなしている人から直接聞けるのが一番スッと入ってくる感じがあるので、とてもありがたかったです。

## 4. 10/30 Tokyo AI Hackathon

イベントページ: [https://raycast.connpass.com/event/369928/](https://raycast.connpass.com/event/369928/)

### ハッカソンのテーマ

> 🚀 Raycast の AI Extensions と MCP サーバーでデスクトップに AI を統合しよう
> 
> Claude AI を活用した Raycast の拡張機能を作成
> - 日々の作業に溶け込む拡張機能
> - MCP サーバーを活用した拡張機能
> - 繰り返しの作業を自動化できる拡張機能

10/30 の本番では、このテーマのもと、プレイベントで考えたアイデアをもとに実際に Raycast Extension を 1 日で作り切ることにチャレンジしました。

### 4.1 イベントの雰囲気

![ハッカソン当日の様子](/images/20251214/2_hackathon_discussion.jpg)

会場は AWS Startup Loft Tokyoで、参加者の体感3割以上が外国籍、イベント全体が英語で進行されるという、かなりインターナショナルな雰囲気でした。
普段参加している Connpass のテックイベントとはまた違った、ハッカソンならではの緊張感がありました。

全体で 100 人以上いたはずなのでアイデアソンに参加していたのは結構珍しかったみたいでしたが、ぶっつけ本番でこの緊張感だと作り終えるのも怪しかったと思うので、プレイベントに参加しておいて良かったです。

### 4.2 タイムスケジュールと開発環境

当日のタイムスケジュールは以下のような流れでした。細かい時間は多少前後しましたが、15:00の開場から21:45の終了まで、約7時間の長丁場でした。

![ハッカソンのタイムテーブル](/images/20251214/2_hackathon_timeline.png)

16時ごろから本格的に開発がスタートし、19:45の提出締め切りまで黙々とコーディングする時間が続きます。
開発中は Raycast、Supabase、Vercel の各スタッフが会場を回り、必要に応じて技術的なサポートをしてくれました。

参加者には全員に事前に **Claude API の100ドル分のクレジット**（期限付き）が配布されていたので、Claude Code を API モードで実質使い放題という環境が整っていました。
このクレジットは開発する Extension の中でモデルを呼び出す API としても活用できたため、AI 機能をフル活用した Extension 開発が可能でした。

### 4.3 作成した Raycast Extension「Trayce」

#### 開発体制

当日は参加者の半分以上がソロで、他が大体 2 人組というチーム構成でした。
私もソロで参加し、プレイベントで考えたアイデアをもとに開発を進めました。

意識したのは以下の点です。

* 1日で動くものを作るために MVP を決める
* ワンコンセプトで伝わりやすいように欲張らない
* ピッチでグダらないように時間を割いて準備する

Github の issue ドリブンでタスクを「クリップボード履歴の表示」「メモ機能の追加」「AI による生成」などのように区切り、一つずつ確実に動かしながら進めることを心がけました。

#### コンセプト

「作業手順をあとから書こうと思っても、どのコマンド打ったか思い出せない」という経験が誰しもあるかと思います。

Trayce は、**クリップボード履歴から作業手順書を自動生成する** Raycast Extension です。
普段コピーしているコマンドや URL に簡単なメモを足すだけで、Claude が Markdown の手順書に整形してくれます。

使い方はシンプルで、作業中にコピーしたものを Raycast で開いて、各項目に「何のためにやったか」をメモして、生成ボタンを押します。
あとは Claude がいい感じに Markdown の手順書にしてくれるので、それを Notion や Jira に貼り付けて共有できます。

![Trayce のデモ](/images/20251214/2_demo_trayce_full_960.gif)

#### 技術スタックと開発の工夫

コマンドは「手順を作る」と「手順を見返す」の2つだけ、データモデルも必要最小限、状態管理も複雑にせず LocalStorage への読み書きをシンプルにしました。

作っている途中で Raycast の Clipboard History API は一度に 5 件までしか取得できないという制約に気づいたので、あまりイケていないなと思いつつユーザーが Extension 内で複数回履歴を追加読み込みするような UX で実装しました。

:::message
Raycast Extension の作成手順については、別記事「**Raycast Extension 開発のすすめ**」で詳しく解説しています。雛形の作成から Store 公開までの流れを ToDoリストを題材にステップバイステップで説明しているので、ぜひ作ってみてください！
:::

### 4.4 ピッチとファイナリスト発表

締め切り後、まずはそれぞれのテーブルで審査員 1 人に対して順番にピッチを行いました。そこで選ばれた方がファイナリストとして全体の場でプレゼンテーションを行う、という 2 段階の選考プロセスでした。

今回特に印象的だったのは、英語でピッチを行う経験でした。
普段のテック系のイベントでは登壇したとしても日本語でやることがほとんどだったので、限られた時間の中で英語を使って技術的な工夫と価値を端的に伝える難しさを実感しましたが、グローバルなエンジニアが集まる環境で充実感がありました。

![ファイナリスト発表の様子](/images/20251214/2_hackathon_final.jpg)

結果として私はテーブルでの選考を通過できませんでしたが、ファイナリストのプレゼンテーションを見て、デモ動画の作り込みやアイデアの切り口の面白さ、見せ方の上手さは大変勉強になりました。特にストーリーテリングの巧みさやユーザー体験を視覚的に伝える演出は、まだまだ場数を増やすべきだなと感じました。

## 5. 10/31 Raycast Meetup Japan #2 with Raycast

イベントページ: [https://raycast.connpass.com/event/370096/](https://raycast.connpass.com/event/370096/)

10/31 には Raycast Meetup Japan に参加し、ハッカソンとはまた違った形で Raycast のロードマップやコミュニティの雰囲気に触れました。

日本橋の Cybozu オフィスで行われ、Lightning Talk、前日のハッカソン入賞者による Extension のデモセッション、CEO の Thomas 氏らを交えた Fireside Chat、Networking という構成で、約3時間にわたってコミュニティの交流と Raycast の未来について語り合うイベントでした。
(以下、部分的に抜粋)

### 5.1 Fireside Chat

![Fireside Chat の様子](/images/20251214/3_meetup_fireside.jpg)

Fireside Chat では、Raycast コアメンバーが日本コミュニティの活発さについて非常に感謝している様子で、Extensions の開発やフィードバックがプロダクトの進化に大きく貢献しているとのことでした。(日本には世界で3番目にユーザー数が多いそうです)

### 5.2 Windows 版 Raycast のデモ

![Windows 版 Raycast のデモ](/images/20251214/3_meetup_win_demo.jpg)

この記事を書いている時点では既にリリースされていますが、 Meetup では待望の Windows 版 Raycast のデモが披露されました。macOS 版と同様、一部先行した UI/UX を Windows で実現しており、クロスプラットフォーム対応の完成度の高さを感じました。

なんといっても CEO によるデモを見れる機会はなかなか無いのでテンション上がりました。

Windows 版のリリースによって Raycast のユーザーベースがさらに拡大することで、コミュニティがより活発になり、新しいアイデアや Extensions が生まれることを期待したいですね！

### 5.3 Networking での出会い (台湾コミュニティメンバーとの交流など)

Networking の時間では、様々なバックグラウンドを持つ参加者と交流する機会がありました。

個人的に良かったのは、台湾の Raycast Community の主催メンバーとの出会いです。

前日のハッカソンでファイナリストのデモ動画がプロ並みのクオリティで印象に残っていたので、勇気を出して話しかけてみたところ、**Screen Studio** という macOS 向けのスクリーンレコーダーを使っていることを教えてくれました。
このアプリは自動的にカーソルにズームしたり、動きを滑らかにしたりして、プロフェッショナルな見た目の動画を数分で作れるとのことでした。
Stripe、Vercel、Google などの企業でも使われているそうです。次回のハッカソンやプロダクト開発ではぜひ活用してみたいと思いました。

@[card](https://screen.studio/)

会話を続けるうちに、この方たちが台湾の Raycast Community の主催メンバーだったことを知り、台湾でも活発にコミュニティ活動が行われていることを知れました。
エンジニア同士で、仕事以外でも Raycast を通じて国境超えたコミュニティと繋がれたことはよかったです。

コミュニティ全体の温度感としては、単に便利なツールを使うだけでなく、自分たちで拡張して価値を生み出していこうという前向きなエネルギーを感じました。Raycast チームの方々も非常にオープンで、フィードバックを積極的に受け入れる姿勢が印象的でした。

## 6. イベントに参加してみて

3 日間のイベントを通して、プロダクト開発やコミュニティへの向き合い方について考えさせられることが多くありました。

私はほぼ Clipboard History を使っているだけのユーザーでしたが、自分の作業効率の改善を Raycast を通じて考えられるようになりましたし、やはり使う側から作る側に回ると、ツールの見え方が変わります。
プレイベントでアイデアを壁打ちし、ハッカソンで作り切り、Meetup で海外のメンバーと繋がる。ユーザーが作り、フィードバックし、知見を共有するサイクルがプロダクトを強くしていて、Raycast チームがコミュニティを大事にしている理由がわかりました。

Raycast Community Japan 主催のイベントはまた今後も行われると思うので、皆さんもこういった機会があればぜひ参加してみてください！
